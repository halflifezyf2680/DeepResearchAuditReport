# Hermes Agent v0.11.0 追踪核实：10 天后，它进化了吗？

## 背景：10 天前我们做了什么

2026 年 4 月中旬，Hermes Agent 以"自我进化的 AI Agent"之名席卷开源社区——公开仅两个月拿下 81,000 颗 Star，被称为"GitHub 增长最快的 Agent 项目"。Nous Research 背后是 6,500 万美元融资，Paradigm（加密 VBA）领投 5,000 万。

我们当时对这个项目做了一次完整的源码审计，逐行读了它的主仓库和自进化子仓库，发现了一个令人不安的事实：**这套"自我进化"系统在工程上根本站不住脚**。10 项核心缺陷全部经源码核实，无一修复——包括词袋重叠当评估指标、进化 Pipeline 大概率是 no-op、Skill 索引全量注入吞噬 token 预算、Agent 可以未经确认直接修改自己的"记忆"、插件系统零安全检查等。

社区贡献者 kirniy 提交了 4 个修复 PR，全部被忽略，未合并。核心团队修复数：0。

完整审计报告见 [report.md](./report.md)，源码级证据见 [hermes-self-evolution-audit.md](../hermes-self-evolution-audit.md)。

---

## 回来看看：10 天后发生了什么

2026-04-23，Hermes Agent 发布了 v0.11.0（代号 "The Interface Release"）。距我们的审计正好 10 天。

10 天里，Hermes Agent 从 81,000 颗 Star 涨到了 112,000 颗，涨了 31,000 颗——比很多项目一辈子的 Star 都多。贡献者从 286 增至 290+。v0.9 到 v0.11.0 期间合并了 761 个 PR，插入了 224,174 行代码。

看起来非常热闹。那之前审计的 10 项核心缺陷呢？

**答案是：核心缺陷几乎纹丝不动。**

最让人意外的变化不是"修了什么"，而是"删了什么"——**整个自进化模块（evolution/）从主仓库消失了**。不是修好了，是连同它的缺陷一起被清理掉了。README 第一行还是 "The self-improving AI agent built by Nous Research"，Star 还在涨，但"自我进化"的代码已经不在了。

以下是逐项核实。

## "自我进化"的下场：代码直接删除

最讽刺的变化：**evolution 模块从主仓库消失了**。

- `evolution/core/fitness.py`（词袋评分）不存在了
- `evolution/skills/evolve_skill.py`（GEPA no-op pipeline）不存在了
- `evolution/` 整个目录不存在了
- hermes-agent-self-evolution 子仓库也找不到了

"self-improving AI agent" 的核心代码被删除了。不是因为修好了，是因为整个模块连同它的缺陷一起被清理掉了。现在社区 feature request（#10354 "Living System Architecture"、#11604 "Hermes Living Subsystem Framework"）仍然 OPEN，标签 P3，说明自进化变成了"计划中"。

但 README 第一行还是写着 "The self-improving AI agent built by Nous Research"。112,000 颗 Star 也是。

## 10 项缺陷逐项核实

### 缺陷 1：Skill 索引全量注入

**判定：部分修复**

新增了平台和工具集的条件过滤（`_skill_should_show()`），不再注入不兼容平台的 skill。但没有任务相关性过滤——不管你在做什么任务，所有兼容平台的 skill 索引仍然全量注入。

Issue #4379（73% 固定开销）、#6839（Lazy Tool Schema Loading）、#13332（Hybrid Pre-Selection）全部 OPEN 无实现。社区提出的 RAG 式语义预选方案无人接手。

### 缺陷 2：自修改无确认

**判定：未修复**

`prompt_builder.py` line 176 的 "patch it immediately -- don't wait to be asked" 原封不动。v0.11.0 新增了 `skills.guard_agent_created` 安全扫描器，但默认关闭，注释自己说"关了因为 agent 本来就能通过 terminal() 执行同样的代码"。

### 缺陷 3：评分是词袋重叠

**判定：代码已删除**

`evolution/` 目录从主仓库移除，`fitness.py` 不再存在。缺陷不再可触达，但也没有被真正修复。自进化变成了"计划中的功能"。

### 缺陷 4：进化 Pipeline no-op

**判定：代码已删除**

self-evolution 子仓库已不存在。Feature request #10354、#11604 仍 OPEN，均为 P3 标签。同缺陷 3。

### 缺陷 5：记忆系统是结构化日志

**判定：部分修复**

新增多 memory provider（Honcho、RetainDB、Holographic）、session search、hindsight metadata。但核心机制仍是"持久化文本 + 回注 prompt"，Reddit 的评价（96% 支持率）"saves notes to a file and reads them next time"在根本上仍然成立。

### 缺陷 6：Prompt 注入防护缺失

**判定：无变化**

evolution 模块已删除使得此缺陷不再可触达，但主仓库插件系统（`importlib` 直接加载）仍无安全扫描。

### 缺陷 7：Pipeline 可靠性缺失

**判定：代码已删除**

随着 evolution 模块移除，此缺陷不再适用。

### 缺陷 8：插件系统零安全检查

**判定：未修复**

v0.11.0 扩展了插件系统（可注册命令、阻止工具执行、重写工具结果），但安全扫描机制仍为 opt-in。插件 attack surface 扩大但防护未跟进。

### 缺陷 9：Skill Hub 信任策略不对等

**判定：未修复**

`skills.guard_agent_created` 新增但默认关闭。builtin skill 仍然完全不扫描。

### 缺陷 10：API Server 无认证

**判定：未修复**

本地 API Server 仍无认证。Issue #2512（Native Windows Support）仍 OPEN。

## 这 10 天到底做了什么？

v0.11.0 的 761 个合并 PR 主要集中在：

1. **React/Ink TUI 完全重写**（~310 次提交）：多行编辑、斜杠命令补全、流式输出、状态栏、子代理覆盖层
2. **Transport ABC 抽象层**：从 `run_agent.py` 中提取 4 种 Transport 实现（Anthropic、ChatCompletions、ResponsesApi、Bedrock）
3. **5 个新 LLM provider**：NVIDIA NIM、Arcee AI、Step Plan、Google Gemini CLI OAuth、Vercel ai-gateway
4. **QQBot**：第 17 个消息平台
5. **插件系统扩展**：可注册命令、阻止工具执行、重写工具结果
6. **`/steer` 命令**：运行中引导 agent
7. **TOOL_USE_ENFORCEMENT_GUIDANCE**：告诉 agent 必须用工具不能只说不做

**没有做的**：token 效率优化（全量注入架构不改）、自修改确认机制（原文不改）、任何架构级债务偿还。

## 发布节奏

47 天 10 个版本，v0.11.0 单版 224,174 行插入。v0.11.0 窗口内有 482 个 bug fix PR。项目在横向扩张（更多平台、更多 provider、更花哨的 UI）的同时，纵向债务（token 开销、架构合理性、自进化实现）持续累积。

---

## 整体判断

Hermes Agent 的能量完全投入在了**横向扩张**上——更多平台、更多 provider、更花哨的 UI。所有与纵向质量相关的 Issue（#4379、#6839、#11115、#13332、#11425）全部保持 OPEN，无任何实现。

核心缺陷的"处理方式"是删除自进化模块，而非修复。自进化代码消失了，但 "self-improving AI agent" 的营销叙事不变。这让营销与代码现实之间的差距进一步扩大。
