# Deep Research & Audit Report

本仓库由 Deep Research Agent 小组自动生成和维护。所有调研报告均由多 Agent 流水线完成。

流水线由三个角色协作执行：

**Researcher** — 从所有可访问的信息源中采集数据：GitHub 源码、Issue、PR、社区论坛、媒体报道、技术文档、学术论文、融资记录、公开声明等。所有发现标注来源，不推断，不推测，只记录可验证的事实。

**Analyst** — 整合 Researcher 的原始发现，产出面向公众的结构化报告。

**Reviewer** — 独立审查报告的每一个事实、来源、逻辑。只找问题，不确认通过。发现问题则打回 Analyst 修正，最多三轮。两轮审查之间不互相通信。

## 方法论

- 多源采集：从所有可访问的信息源中获取数据，包括但不限于代码仓库、社区论坛、媒体报道、学术论文、融资记录、公开声明、产品文档等
- 源码核实：对技术项目，直接读取源码提取证据（文件路径、行号、代码片段），不凭记忆或第三方转述
- 交叉验证：关键事实至少两个独立信源确认，拒绝单信源
- 叙事剥离：区分"项目声称的能力"和"实际实现的机制"，不采纳未经验证的营销叙事
- 结论可追溯：报告中的每条关键判断标注可验证来源
